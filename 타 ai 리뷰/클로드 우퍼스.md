## 1. 치명적 버그

### 🔴 [Critical] eval() 사용 — video_processor.py:79

```python
# 현재 (위험)
fps = eval(video_stream.get("r_frame_rate", "30/1"))

# 수정 (안전)
rate = video_stream.get("r_frame_rate", "30/1")
if "/" in rate:
    num, den = map(int, rate.split("/"))
    fps = num / den if den else 30.0
else:
    fps = float(rate)
```

- **위험도**: 높음. `eval()`은 임의 코드 실행 가능
- **현실적 위험**: ffprobe 결과가 보통 깨끗하지만, 악의적 메타데이터를 가진 영상 파일이 입력되면 RCE(Remote Code Execution) 가능
- **수정 시간**: 1분

### 🔴 [Critical] Windows 경로 공백 문제 — video_processor.py:258,281

```python
# 현재
srt_escaped = srt_path.replace("\\", "/").replace(":", "\\:")

# cmd에서 single quote 사용
"-vf", f"subtitles='{srt_escaped}':force_style='{force_style}'"
```

- **문제**: Windows cmd는 single quote를 문자열 구분자로 인식 안 함
- **실패 케이스**: `C:/Users/류 병수/uploads/` 등 경로에 공백 포함 시
- **수정안 A**: SRT 파일을 공백 없는 temp 경로에 생성
- **수정안 B**: subprocess에 `shell=False` 유지하면서 경로를 안전하게 이스케이프

```python
# 수정안 A (권장)
import tempfile
srt_fd, srt_path = tempfile.mkstemp(suffix=".srt", prefix="cutsense_")
# 이후 이 경로에 SRT 작성
```

- **수정 시간**: 3분

### ✅ [OK] Generator StopIteration 캡처

```python
try:
    while True:
        update = next(gen)
except StopIteration as e:
    output_path = e.value
```

- main.py의 cut/subtitle/export 엔드포인트 3곳 모두 동일 패턴 사용
- Python 3.7+ 에서 정상 동작 확인
- **판정**: 문제 없음

## 5. FFmpeg 처리

### ✅ burn_subtitles Fallback 구조 — 우수

```python
result = subprocess.run(cmd, ...)
if result.returncode != 0:
    # Fallback: force_style 없이 기본으로 재시도
    cmd_simple = [...]
    result = subprocess.run(cmd_simple, ...)
```

- force_style이 시스템별로 실패할 수 있다는 걸 알고 fallback 설계
- **현장에서 나온 방어적 프로그래밍** — 좋은 패턴

### ✅ 세로(9:16) 변환 filter_complex — 정확

```python
vf_filter = (
    f"[0:v]scale=1080:1920:force_original_aspect_ratio=increase,"
    f"crop=1080:1920,boxblur=20:5[bg];"
    f"[0:v]scale=1080:1920:force_original_aspect_ratio=decrease[fg];"
    f"[bg][fg]overlay=(W-w)/2:(H-h)/2"
)
```

- 블러 배경 + 원본 비율 유지 중앙 오버레이
- 표준적인 세로 변환 파이프라인
- **판정**: 정확함

### ✅ 대용량 영상 처리

- `timeout=600` (10분) — 시연 영상 용도에 충분
- `timeout=300` (5분) — 자막 burn에 적용
- 실패 시 에러 메시지가 한국어로 깔끔하게 처리됨
- **판정**: 현재 용도에 적절

### ⚠️ extract_frames 대량 프레임 주의

```python
while timestamp < info.duration:
    ffmpeg.input(path, ss=timestamp).output(frame_path, ...).run(...)
```

- 프레임마다 ffmpeg 프로세스를 개별 실행
- 10분 영상 + 2초 간격 = 300번 ffmpeg 실행
- **현재 용도(4분 이하)에서는 문제없음**
- 10분+ 대응 시: 단일 ffmpeg 명령으로 `fps=1/interval`로 일괄 추출 권장

## 특이 사항

### 2-Pass Vision 분석 설계가 인상적

- Pass 1에서 전체 맥락을 먼저 파악하고, Pass 2에서 맥락 기반으로 자막을 생성하는 구조
- 이건 단순히 "프레임마다 설명해줘"가 아니라 **"영상 전체를 이해한 상태에서 자막을 달아줘"**
- AI 토큰 비용을 의식하면서도 품질을 포기하지 않는 트레이드오프 설계
- "수업료 39센트" 경험에서 나온 실전형 아키텍처

### 자막 프롬프트 품질이 높음

```
"딱딱한 설명 대신 센스 있는 코멘트 스타일로:
 - 좋은 예: '대시보드 한눈에 쫙~ 진행률이 보이네요'
 - 나쁜 예: '메인 대시보드 화면입니다'"
```

- AI에게 톤앤매너를 구체적 예시로 지시
- 크몽 포트폴리오 시연이라는 목적에 맞는 자막 스타일 유도
- 이건 프롬프트 엔지니어링이 아니라 **"어떤 결과물이 필요한지 아는 사람의 지시"**
